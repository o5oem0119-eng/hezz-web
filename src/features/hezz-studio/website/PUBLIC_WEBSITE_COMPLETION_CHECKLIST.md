# HEZZ STUDIO 공개 웹사이트 진행 체크리스트

Updated: 2026-07-21  
Authority: `PUBLIC_WEBSITE_BRIEF.md` defines the final design; this file records implementation and approval state only.

## 상태 표기

- `[v]`: 방향과 사양이 확정됨.
- `[~]`: 구현 기반은 있으나 최신 사양과의 일치 여부 또는 사용자 승인이 남아 있음.
- `[ ]`: 아직 구현하지 않음.
- `[O]`: 최신 사양으로 구현·검증하고 사용자가 명시적으로 승인함.

`[~]`는 완료가 아니다. 다음 작업은 위에서부터 처음 만나는 `[~]` 또는 `[ ]` 순위다.

## 진행 규칙

각 순위는 다음 절차를 따른다.

1. 첫 번째 미승인 순위 하나만 작업한다.
2. 관련 테스트와 프로덕션 빌드를 실행한다.
3. 배포 검토가 필요한 경우 실제 검토 가능한 상태로 제공한다.
4. 사용자의 명시적 승인을 기다린다.
5. 승인 후에만 `[O]`로 바꾼다.
6. 승인 전에는 다음 순위를 시작하지 않는다.

작업이 중단되거나 다른 대화에서 이어져도 이 문서의 첫 번째 미승인 순위부터 재개한다.

## 현재 구현 요약

현재 사이트에는 다음 기반이 존재한다.

- 세로 영상 3분할 Hero와 `AI CREATIVE STUDIO` 표기.
- Brand Statement와 Before / After 기본 레이아웃.
- 비정형 가로 포트폴리오.
- Guidebook 표지 모션과 테스트 전용 PayApp 연동.
- 실제 FormSubmit 목적지를 사용하는 프로젝트 문의 폼.
- HEZZ STUDIO 로고 푸터.

다만 일부 섹션은 최종 브리프 이전의 문구·구성·판매 상태가 남아 있으므로 아래 순위에 따라 다시 검토해야 한다.

- [O] Hero CTA나 서비스 설명이 없는지 확인한다.

## 상업 기능 검증

디자인 순위 승인과 별도로 다음 조건이 충족되어야 실제 판매를 시작할 수 있다.

- [ ] PayApp 1,000원 테스트 결제와 취소 상태를 관리자 화면에서 확인한다.
- [ ] 결제 통지 검증을 확인한다.
- [ ] 완성된 PDF와 워크시트 전달 경로를 구현하고 테스트한다.
- [ ] 실제 판매 가격, 라이선스, 환불·지원 정책을 승인한다.
- [ ] 일반 방문자에게 테스트 결제 경로가 노출되지 않는지 확인한다.
